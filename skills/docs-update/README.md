# Docs Update

## Purpose
Keep documentation aligned with workflow and architecture changes.

## When to use
- Behavior changed
- Workflow changed
- Public guidance drifted

## When not to use
- No user-visible or maintainer-visible change occurred

## Inputs
- Changed files
- Previous docs
- New behavior

## Outputs
- Updated docs list
- Missing-doc warnings
- Suggested edits

## Non-goals
- Rewriting unrelated docs
- Marketing copy creation
