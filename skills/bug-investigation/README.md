# Bug Investigation

## Purpose
Move from symptom to likely root cause in a structured way.

## When to use
- Failure symptoms exist
- Cause is unclear
- Multiple plausible explanations exist

## When not to use
- Feature planning
- Architecture design without a concrete bug

## Inputs
- Symptom description
- Reproduction clues
- Relevant files or logs

## Outputs
- Likely causes
- Proposed checks
- Most plausible root cause path

## Non-goals
- Pretending certainty without evidence
- Full refactor unrelated to the bug
