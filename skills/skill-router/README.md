# Skill Router

## Purpose
Select the right narrow capability for the current task instead of doing everything ad hoc.

## When to use
- Unsure which skill to invoke first
- A task spans multiple possible audit paths

## When not to use
- You already know the exact skill needed

## Inputs
- Task description
- Primary risk or ambiguity

## Outputs
- Suggested skill order
- Why each skill is appropriate

## Non-goals
- Acting as the skill itself
- Replacing the canonical routing notes in core
