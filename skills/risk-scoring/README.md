# Risk Scoring

## Purpose
Estimate task risk and determine how much structure and verification are needed.

## When to use
- Large changes
- Cross-cutting refactors
- Unclear ownership or weak evidence

## When not to use
- Tiny isolated edits with obvious behavior

## Inputs
- Task scope
- Affected areas
- Verification coverage

## Outputs
- Risk level
- Why the task is risky
- Recommended rigor level

## Non-goals
- Replacing review
- Replacing verification
