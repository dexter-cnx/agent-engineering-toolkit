# Architecture Review

## Purpose
Evaluate boundaries, interfaces, responsibility split, layering, and long-term maintainability.

## When to use
- Structural changes
- Module decomposition
- Boundary or dependency questions

## When not to use
- Pure wording changes
- Trivial fixes with no architectural impact

## Inputs
- Problem statement
- Current structure
- Proposed change

## Outputs
- Boundary assessment
- Risks
- Recommended structural direction

## Non-goals
- Full implementation
- Running project-specific verification commands
