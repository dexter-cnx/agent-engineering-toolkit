# Dependency Review

## Purpose
Review external dependency impact, coupling risk, and adapter needs.

## When to use
- Adding or changing external dependencies
- Evaluating lock-in risk

## When not to use
- Internal refactors with no dependency change

## Inputs
- Dependency name
- Proposed usage
- Affected boundaries

## Outputs
- Risk summary
- Adapter recommendations
- Adoption cautions

## Non-goals
- Package management automation
- Full vendor comparison research
