# Feature Workflow Example

Use the canonical lifecycle from `docs/prompt-pipeline.md`.

## Example sequence
1. Lead frames the task.
2. Architect defines structure.
3. Builder implements.
4. Reviewer critiques.
5. Verifier checks.
6. Finalizer packages the result.
7. Memory stores durable notes.

## Notes
- Use review to critique quality.
- Use verification to assess confidence and evidence.
- Update memory when a decision or constraint should survive the run.
