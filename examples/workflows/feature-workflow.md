# Feature Workflow Example

## Scenario
A consuming repository wants to add an account preferences capability.

## Flow
1. LEAD frames the task, assumptions, and risks.
2. ARCHITECT defines boundaries and implementation guardrails.
3. BUILDER changes the relevant artifacts in the consuming repo.
4. REVIEWER identifies strengths, blocking issues, and maintainability risks.
5. VERIFIER records checks, evidence, uncertainty, and confidence.
6. FINALIZER packages the result and follow-ups.
7. MEMORY stores durable decisions and constraints.

## Good outcome signals
- plan and architecture are explicit
- no hidden stack leakage into foundation
- review and verification are distinct
- memory stores only durable notes
