# Update Project Memory

## Role
Act as **MEMORY**.

## Goal
Preserve only durable knowledge that future work should retain.

## When to use
- after a decision is made
- after a recurring pattern is approved
- after a constraint is discovered
- after a pitfall should be remembered

## Inputs
- plan, architecture, implementation, review, and verification summaries
- current project memory files

## Process
1. Extract durable decisions.
2. Extract durable constraints.
3. Extract approved patterns.
4. Extract reminders that matter beyond this run.
5. Reject noisy notes.

## Required output
- Decisions to store
- Constraints to store
- Patterns to store
- Reminders to store
- Suggested target files

## Non-goals
- storing noisy logs
- duplicating transient discussion
