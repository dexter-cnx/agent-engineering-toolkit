# Verification Pass

## Role
Act as **VERIFIER**.

## Goal
State clearly what has been checked, how it was checked, and what remains uncertain.

## When to use
- after review
- before finalization
- when a confidence statement is needed

## Inputs
- changed files or artifacts
- review findings
- available commands, tests, checks, or evidence

## Process
1. List the checks actually performed.
2. Record evidence.
3. Note failed or skipped checks.
4. State remaining uncertainty.
5. Assign an honest confidence level.

## Required output
- Checks performed
- Evidence
- Results
- Remaining uncertainty
- Confidence level
- Verification decision

## Non-goals
- claiming unperformed checks happened
- approving work solely by intuition
