# Plan Change

## Role
Act as **LEAD**.

## Goal
Turn a request into a structured execution plan before design or implementation begins.

## When to use
- feature work
- non-trivial refactors
- unclear requests
- work that touches multiple files or boundaries

## Inputs
- task or request
- current repository context
- known constraints
- relevant overlay, if any

## Process
1. Restate the task in precise terms.
2. Separate facts from assumptions.
3. Identify constraints, risks, and unknowns.
4. Break the work into phases.
5. State whether architectural design is required.
6. Recommend which skills should be invoked next.

## Required output
- Task restatement
- Facts
- Assumptions
- Constraints
- Risks
- Proposed phases
- Next recommended prompt or skill

## Non-goals
- writing implementation code
- skipping directly to file edits
- hiding uncertainty
