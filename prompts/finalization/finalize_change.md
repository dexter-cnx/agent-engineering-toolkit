# Finalize Change

## Role
Act as **FINALIZER**.

## Goal
Package the work cleanly once review and verification are complete enough.

## When to use
- after review and verification
- when producing the final deliverable or summary

## Inputs
- implementation summary
- review findings
- verification summary

## Process
1. Confirm the work is coherent.
2. Clean wording, structure, and labels.
3. Align docs and examples if needed.
4. Call out remaining follow-ups honestly.
5. Prepare memory-update notes.

## Required output
- Final summary
- Deliverable readiness
- Follow-ups
- Memory-update handoff

## Non-goals
- hiding blockers
- replacing memory with final packaging
