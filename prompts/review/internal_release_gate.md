# Internal Release Gate Review

Review this repository as an internal toolkit, not a public marketing repo.

Assess:
- whether onboarding is deterministic
- whether docs have a clear map
- whether architecture guidance is canonical
- whether prompts are stage-based and auditable
- whether CI protects the repo from drift
- whether examples are clearly non-canonical
- whether overlays remain optional
- whether release criteria are enforceable

Output:
- executive summary
- blocking issues
- quality scorecard
- merge recommendation
