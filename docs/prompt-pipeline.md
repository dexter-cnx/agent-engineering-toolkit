# Prompt Pipeline

This repository includes a stage-oriented prompt set.

## Core prompts
- `plan_change.md`
- `architecture_review.md`
- `implement_change.md`
- `review_change.md`
- `verification_pass.md`
- `finalize_change.md`
- `update_project_memory.md`

## Additional prompts
- `audit_repo.md`
- `investigate_bug.md`
- `scaffold_overlay.md`
- `adopt_toolkit_in_repo.md`

## Recommended order
For a non-trivial feature:

1. `plan_change.md`
2. `architecture_review.md`
3. `implement_change.md`
4. `review_change.md`
5. `verification_pass.md`
6. `finalize_change.md`
7. `update_project_memory.md`

## Notes
- not every task needs every prompt
- meaningful structural work usually benefits from all major stages
- project memory should be updated whenever a durable decision is made
