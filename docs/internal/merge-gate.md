# Internal Merge Gate

Before merging, verify all of the following:

- The change has one clear owner doc for each concept.
- New docs are linked from the right navigation page.
- New prompts do not redefine architecture that already exists elsewhere.
- Examples remain examples and are labeled as such.
- Tutorials point to canonical references when teaching policy.
- There is a concrete verification path.
- CI checks pass.
- Review templates were run for non-trivial changes.

Recommended labels:
- docs-only
- architecture-impact
- prompt-impact
- tutorial-impact
- release-impact
