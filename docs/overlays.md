# Overlays

## Purpose
Overlays let the toolkit stay general while still supporting real stacks.

## Included overlays

### mobile-flutter
For mobile and Flutter-heavy repositories.

### backend-node
For Node, Express, Nest, API services, and integration backends.

### web-frontend
For web UI, dashboard, product surfaces, design systems.

### python-service
For Python services, FastAPI, automation, adapters, AI services.

## How to use an overlay
1. Start from foundation.
2. Copy or reference the relevant overlay.
3. Merge overlay assumptions into the consuming repo.
4. Add project-specific rules on top.

## Important
An overlay should not rewrite the identity of the foundation.
It should extend it.
