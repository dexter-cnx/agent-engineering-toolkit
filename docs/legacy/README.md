# Legacy Surface Archive

This directory stores non-canonical root documents retained for historical compatibility and migration context.

## Purpose

- Preserve previously root-level onboarding and index materials without exposing them as canonical entry points.
- Keep repository root focused on canonical Agent Engineering OS surfaces.
- Provide a stable archive location for legacy references used by older documentation and external links.

## Governance

- Canonical onboarding remains: `README.md -> docs/get-started.md -> docs/adoption-paths.md -> docs/overlays.md`.
- New onboarding or governance guidance must not be added back to root legacy filenames.
- Legacy files may be consulted for history, but canonical updates must happen in canonical docs.
