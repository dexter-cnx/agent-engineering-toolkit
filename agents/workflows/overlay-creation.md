# Workflow: Overlay Creation

## Mission
Create a specialization overlay that extends, but does not replace, root guidance.

## Inputs
Target stack/domain, root policy contracts, overlay template.

## Outputs
- overlay README with use/boundary/non-replacement statements
- specialization docs and examples
- passing overlay governance checks

## Boundaries
- No root identity rewrites from overlay scope.
- No duplication of canonical onboarding inside overlay docs.

## Escalation conditions
- overlay requires violating root stack-neutral constraints
- duplicate guidance between root and overlay cannot be resolved
