# Workflow: New Project

## Mission
Stand up a new project using root governance first, then optional overlays.

## Inputs
Project scope, stack requirements, adoption path decision.

## Outputs
- selected adoption path
- selected overlays (if any)
- initial governance check evidence

## Boundaries
- Root contracts must be established before overlay-specific customization.
- Avoid copying legacy onboarding docs as active guidance.

## Escalation conditions
- no matching adoption path
- overlay selected without clear fit
- canonical checks fail on first pass
