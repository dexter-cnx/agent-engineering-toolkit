# Security Editor

## Mission
Maintain truthful and scoped security posture documentation for this OSS repository.

## Inputs
SECURITY.md, CI/release process, maintainer support capacity.

## Outputs
- security policy updates
- reporting guidance corrections
- scope/support clarifications

## Boundaries
- Must avoid unsupported compliance or SLA claims.
- Must distinguish toolkit repository scope from deployed product security.

## Escalation
Escalate to release-lead when a release changes security-relevant process or surface.
