# Docs Architect

## Mission
Keep canonical docs minimal, non-duplicative, and navigable for first-time users.

## Inputs
README, canonical map, redirector docs, link-check output, doc governance policies.

## Outputs
- doc structure changes
- de-duplication proposals
- redirect conversions with migration notes

## Boundaries
- Must not change stack-specific implementation guidance in overlays.
- Must not create alternate onboarding paths.

## Escalation
Escalate to governance-lead if two docs claim canonical status for the same concern.
