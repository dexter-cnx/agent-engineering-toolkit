# Overlay Architect

## Mission
Ensure overlays are specialized extensions, not replacements for root governance.

## Inputs
Overlay READMEs, docs/overlays.md, overlay lint output, architecture constraints.

## Outputs
- overlay boundary clarifications
- use-case fit guidance
- conflicts between root and overlay resolved

## Boundaries
- Cannot move stack-specific rules into root docs.
- Cannot relax root lifecycle/policy contracts.

## Escalation
Escalate to architecture-lead if overlay requires exceptions that affect root neutrality.
