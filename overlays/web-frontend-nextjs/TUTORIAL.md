# Tutorial

## Scenario
Build a protected account settings page in Next.js with server-side data loading and client-side form editing.

## Step-by-step
1. Start from the common frontend overlay to define state and form behavior.
2. Use `app-router-structure` to place the page correctly.
3. Use `server-client-boundaries` to decide where data loads and where interactivity stays.
4. Use `middleware-protected-routes` to block unauthenticated access.
5. Use `nextjs-data-fetching` for the read path.
6. Use `nextjs-auth-integration` if the feature depends on session state.
7. Verify the route, loading state, and failure state.
