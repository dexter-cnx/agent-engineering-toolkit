# How to Use This Overlay

## What it is
This overlay gives you the Next.js-specific implementation layer for frontend work.

## When to use skills
Use a skill when the task depends on App Router, server/client boundaries, route handlers, middleware, or auth integration.

## Minimal onboarding
1. Read `README.md`
2. Read `overlays/web-frontend-common/README.md`
3. Pick the Next.js-specific skill
4. Use the matching prompt
5. Check the example for the expected structure

## Common tasks
| Task | Skill | Prompt |
| --- | --- | --- |
| Create a new App Router page | `app-router-structure` | `prompts/generate_nextjs_feature.md` |
| Split server and client logic | `server-client-boundaries` | `prompts/refactor_feature.md` |
| Add a route handler | `route-handlers` | `prompts/generate_nextjs_feature.md` |
| Protect a route | `middleware-protected-routes` | `prompts/review_security.md` |
| Add data fetching | `nextjs-data-fetching` | `prompts/new_feature.md` |
| Add auth integration | `nextjs-auth-integration` | `prompts/connect_frontend_to_backend.md` |
