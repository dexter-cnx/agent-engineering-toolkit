# Shared Skill Contract

Use this contract for atomic Next.js frontend skills in this overlay.

## Every skill must define
- purpose
- use when
- do not use when
- inputs required
- outputs expected
- workflow steps
- validation checklist
- references to prompts or examples

## Boundary rules
- keep App Router placement in the right skill
- keep server/client decisions explicit
- keep route handlers thin
- keep data fetching close to the owning route or feature
