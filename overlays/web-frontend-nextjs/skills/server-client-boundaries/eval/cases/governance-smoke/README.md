# Governance Smoke Case

This static case keeps the server/client boundary skill aligned with the Karpathy plane.

## Scenario

A Next.js feature needs a clear split between server-only data loading and client-only
interactivity.

## Checkpoints

- the contract file exists
- the expected result fixture exists
- the skill can be evaluated with the canonical Karpathy scripts
- promotion stays regression-safe and token-governed
