# Examples

Worked examples for the agent-friendly CLI overlay.

Use these as short references for recurring workflows that should stay
read-first, compact, and safe by default.

## Recommended reading order

1. [CLI adoption workflow](./cli-adoption-workflow.md)
2. [CLI operator workflow](./cli-operator-workflow.md)
3. [Dry-run versus write](./cli-dry-run-vs-write.md)
4. [Large export flow](./cli-export-large-payload.md)

6. [Failure handling](./cli-failure-handling.md)

## Language versions

- English files use the current names in this folder.
- Thai companions use the same basename with `_TH.md`.
- The pair for each scenario should stay structurally aligned.
- The Thai hub is [README_TH.md](./README_TH.md).

## What these examples cover

- how to turn recurring work into a reusable CLI
- how to use discovery before exact reads
- how to keep large outputs in files
- how to separate draft or dry-run paths from live writes
- how to verify the CLI from outside the source directory
- how to report failures clearly

## Scope

These examples are intentionally stack-neutral.
Use repository-specific overlays for language, framework, or product details.
