# Governance Smoke Case

This static case keeps the Python service boundary skill aligned with the Karpathy plane.

## Scenario

A Python service needs a clear split between routers, orchestration, persistence, adapters,
and domain or schema boundaries.

## Checkpoints

- the contract file exists
- the expected result fixture exists
- the skill can be evaluated with the canonical Karpathy scripts
- promotion stays regression-safe and token-governed
