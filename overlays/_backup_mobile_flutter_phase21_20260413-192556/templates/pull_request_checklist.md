# Flutter PR Checklist

- [ ] Architecture follows repo policy
- [ ] No business logic introduced in widgets
- [ ] Localization updated for new user-facing strings
- [ ] Unit/widget/integration tests updated where behavior changed
- [ ] New dependencies justified
- [ ] Build/run/test commands verified
- [ ] Platform setup documented if iOS/Android/Web behavior changed
- [ ] Screens reviewed for accessibility and responsive layout basics
