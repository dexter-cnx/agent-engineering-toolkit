# Shared Skill Contract

Use this contract for atomic frontend skills in this overlay.

## Every skill must define
- purpose
- use when
- do not use when
- inputs required
- outputs expected
- workflow steps
- validation checklist
- references to prompts or examples

## Boundary rules
- keep framework-specific routing out of common skills
- keep backend API design out of common skills
- keep reusable UI patterns separate from page orchestration
- keep tests oriented around behavior
