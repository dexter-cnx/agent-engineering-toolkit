# Repo Adaptation Checklist

Use this checklist when copying the overlay into a real web frontend repository.

## Decide
- framework
- routing model
- state model
- data layer
- auth model
- i18n strategy
- test strategy
- deployment target

## Verify
- folder structure matches the chosen stack
- page boundaries stay explicit
- design-system rules are documented
- release checks are wired
- project memory captures durable decisions
