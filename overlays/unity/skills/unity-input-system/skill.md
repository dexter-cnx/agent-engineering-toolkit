# unity-input-system Skill

Use this skill when input binding, rebinding, or device support needs a clear Unity convention.

## Rules

- Keep input actions separate from gameplay logic.
- Organize action maps around user intent and device coverage.
- Treat rebinding as a first-class workflow, not a later patch.
- Support keyboard, mouse, gamepad, and touch through a single abstraction layer where possible.
- Keep input assets and generated wrappers reviewable and easy to regenerate.

## Deliverables

- action-map plan
- rebinding strategy
- device coverage notes
- validation checklist
