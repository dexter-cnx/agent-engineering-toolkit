# unity-package-management Skill

Use this skill when package manifests, locks, or dependency strategy need review.

## Rules

- Pin packages intentionally and review upgrades as change requests.
- Keep git dependencies and local packages explicit in the manifest.
- Treat the lock file as part of the reproducible build surface.
- Review package risk before adopting new dependencies.
- Avoid hidden version drift across contributors or CI.

## Deliverables

- package plan
- version pinning notes
- dependency risk notes
- verification checklist
