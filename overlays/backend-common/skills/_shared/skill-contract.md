# Shared Skill Contract

Use this contract for atomic backend skills in this overlay.

## Every skill must define
- purpose
- use when
- do not use when
- inputs required
- outputs expected
- workflow steps
- validation checklist
- references to prompts or examples

## Boundary rules
- keep transport framework details out of common skills
- keep persistence implementation out of common skills
- keep auth shape and permission shape explicit
- keep error handling and validation visible
