# บทเรียนการใช้งาน

## สถานการณ์
ออกแบบ endpoint สำหรับอัปเดตโปรไฟล์ผู้ใช้ พร้อม validation, audit logging และ permission check

## ขั้นตอนทีละขั้น
1. เริ่มจาก `api-contracts`
2. ใช้ `validation-error-handling`
3. ใช้ `role-permission-model`
4. ใช้ `backend-testing-strategy`
5. ตรวจ contract และ failure behavior แบบ end-to-end
