# How to Use This Overlay

## What it is
This overlay gives you the backend concepts that should stay stable across runtimes.

## When to use skills
Use skills when the task is about contracts, validation, auth shape, permissions, file safety, email flows, or backend testing strategy.

## Minimal onboarding
1. Read `README.md`
2. Open `SKILLS_INDEX.md`
3. Identify the backend problem
4. Choose one skill
5. Use the matching prompt file

## Common tasks
| Task | Skill | Prompt |
| --- | --- | --- |
| Define an API contract | `api-contracts` | `prompts/generate_api_contract.md` |
| Plan auth/session shape | `auth-token-concepts` | `prompts/new_feature.md` |
| Design refresh behavior | `refresh-token-strategy` | `prompts/new_feature.md` |
| Model roles and permissions | `role-permission-model` | `prompts/review_security.md` |
| Design a CRUD resource | `crud-resource-design` | `prompts/generate_api_contract.md` |
| Secure file uploads | `file-handling-safety` | `prompts/review_security.md` |
