# Governance Smoke Case

This static case keeps the skill contract, example, and Karpathy run path aligned.

## Scenario

A Flutter app needs a governed HTTP client boundary with typed DTOs, interceptor setup,
auth header handling, retry policy, and error mapping.

## Checkpoints

- the skill contract is present
- the expected result fixture is present
- the skill can be evaluated with the canonical Karpathy scripts
- promotion stays regression-safe and token-governed
