# Implementation Checklist

- [ ] Confirm the feature outcome that requires `flutter-build-flavors`.
- [ ] Confirm which companion skills are also needed.
- [ ] Define capability boundaries before generating code.
- [ ] Keep provider or plugin integration behind an adapter, service, or repository boundary.
- [ ] Keep UI code free from business orchestration and provider-specific parsing.
- [ ] Define typed inputs, outputs, and error states.
- [ ] Document platform notes when Android, iOS, or web behavior differs.
- [ ] Update overlay-facing documentation if the usage model changes.
