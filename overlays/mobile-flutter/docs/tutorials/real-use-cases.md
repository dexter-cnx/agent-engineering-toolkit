# Real Use Cases

## Build a new Flutter app with AI

1. Start with `workflows/new-project/README.md`.
2. Scaffold architecture.
3. Select Riverpod or GetX state.
4. Add routing and localization.
5. Finish with release readiness and CI.

## Add a feature using a skill

1. Audit the feature boundary.
2. Scaffold the feature folder.
3. Add state and repository wiring.
4. Add tests.

## Fix an architecture violation

1. Use `flutter-clean-architecture-audit`.
2. Identify the layer violation.
3. Move the code to the correct boundary.
4. Verify the output tree.

## Release an app

1. Use the release workflow.
2. Run Android signing readiness.
3. Run iOS readiness.
4. Run performance and CI checks.
