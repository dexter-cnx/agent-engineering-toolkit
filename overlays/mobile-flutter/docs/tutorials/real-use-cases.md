# Real Use Cases

## Build a new Flutter app with AI

1. Start with `workflows/new-project/README.md`.
2. Scaffold folders with `flutter-feature-folder-scaffold`.
3. Create domain contracts with `flutter-feature-contract-scaffold`.
4. Pick `flutter-riverpod-state-skeleton` or `flutter-getx-controller-skeleton`.
5. Declare routes with `flutter-go-router-route-map`.
6. Add redirects with `flutter-go-router-redirect-guard`.
7. Wire deep links with `flutter-go-router-deeplink-wireup`.
8. Sync localization with `flutter-localization-csv-sync`.
9. Finish with release readiness and CI.

## Add a feature using a skill

1. Audit the feature boundary.
2. Scaffold the feature folder.
3. Add domain contracts.
4. Add the state skeleton.
5. Add repository wiring.
6. Add tests.

## Fix an architecture violation

1. Use `flutter-clean-architecture-audit`.
2. Identify the layer violation.
3. Move the code to the correct boundary.
4. Verify the output tree.

## Release an app

1. Use the release workflow.
2. Run Android signing readiness.
3. Run iOS readiness.
4. Run performance and CI checks.
