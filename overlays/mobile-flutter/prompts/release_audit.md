# Prompt: Release Audit

Use the `mobile-flutter` overlay to perform a release-readiness review.

## Load these skills first
- `flutter-android-release-validate`
- `flutter-ios-release-readiness`
- `flutter-web-loading-shell`
- `flutter-performance-audit`

## Also read
- `overlays/mobile-flutter/workflows/release-app/README.md`
- `overlays/mobile-flutter/policies/testing/README.md`
- `overlays/mobile-flutter/policies/secrets/README.md`

## Expected checks
- build sanity
- test coverage for changed behavior
- localization completeness
- platform permission notes
- web loading and performance risks
- crash/error handling paths
- environment or flavor readiness
- rollback or mitigation notes
