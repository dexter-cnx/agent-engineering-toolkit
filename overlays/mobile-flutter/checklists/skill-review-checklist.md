# Maintainer Skill Review Checklist

- [ ] The contribution explains why an existing skill was insufficient.
- [ ] The new skill is atomic and has one clear responsibility.
- [ ] All required schema fields are present and non-empty.
- [ ] The title matches the folder name.
- [ ] The references section links to at least one example and one template.
- [ ] There is no placeholder text.
- [ ] The purpose does not duplicate another active skill.
- [ ] The trigger set does not duplicate another active skill.
- [ ] The workflow is orchestration-only if the file is a workflow.
- [ ] The impact map and index were updated if needed.
- [ ] CI validation passes locally and in the repository workflow.
