# Feature Flow Runbook

Use the canonical lifecycle from `docs/prompt-pipeline.md`.

## Minimum runbook
1. Plan the change
2. Review architecture
3. Implement
4. Review
5. Verify
6. Finalize
7. Update memory

## Runbook notes
- Structural work should include all seven stages.
- Review and verification are not the same step.
- A durable architectural decision should trigger a memory update.
- Link the project-specific verification commands in the consuming repo beside this runbook.
