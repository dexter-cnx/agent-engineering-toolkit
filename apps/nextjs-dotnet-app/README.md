# Next.js + ASP.NET Core App Starter

Canonical split full-stack starter with a Next.js App Router frontend and an ASP.NET Core
backend.

## Layout

- `frontend/` - Next.js App Router client and typed API consumption
- `backend/` - ASP.NET Core API with Domain, Application, Infrastructure, and Web layers

## What it demonstrates

- JWT auth and refresh token flow
- typed client consumption of shared contracts
- posts CRUD path
- FluentValidation and EF Core on the backend
- health endpoint and explicit layering
